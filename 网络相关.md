#### 跨域
```
    起因：
        由于浏览器同源保护策略，协议，域名，端口，三者都相同，否则为跨域。

	常用跨域解决方案：
        JSONP，Proxy代理，nginx反向代理，cors，postMessage，document.domain，upgrade-insecure-requests等
```

#### 从输入url到页面展示到底发生了什么
```
    1. 输入地址
    2. 浏览器拿域名先到本地hosts文件查找ip，找不到会发起DNS请求到DNS服务器去查找ip
    3. 成功获取到ip后向web服务器发送一个http请求，为了防止已经失效的连接请求报文突然又传到服务端，因而产生错误，会先发起TCP三次握手
    4. 服务器处理请求，解析报文，并且返回http响应
        1xx: 信息类  2xx: 成功  3xx: 重定向  4xx: 客户端错误  5xx: 服务端错误
        301：永久重定向
        302：临时重定向(有利于SEO，搜索引擎会抓取新的内容而保存旧的网址)
        401：未授权：登录失败
        403：没有权限访问该页面，禁止访问
        404：找不到页面
        405：请求类型错误(get, post等)
        500：服务器内部错误
        502：服务器已关闭
        504：网关超时
    5. 浏览器渲染页面
        -> 解析html 
        -> 构建dom树
        -> 构建render树
        -> 布局render树
        -> 绘制render树
        回流(reflow): 当盒模型位置、大小等属性改变触发
        重绘(repain): 颜色等改变
```
